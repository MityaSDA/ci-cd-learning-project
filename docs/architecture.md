# Архитектура проекта CI/CD Demo Project

## Обзор

Этот проект — **demo full-stack приложение**, которое демонстрирует:

- разработку frontend с использованием **React**
- разработку backend с использованием **Flask**
- CI/CD пайплайны на базе **GitHub Actions**
- контейнеризацию с помощью **Docker**
- деплой приложения на платформу **Render**
- подход **Documentation as Code** с использованием **MkDocs**

Архитектура намеренно сделана простой, понятной и удобной
для учебных целей и портфолио.

## Компоненты системы

### Frontend

- **Технология**: React (Create React App)
- **Ответственность**:
  - отображение пользовательского интерфейса
  - отправка HTTP-запросов к backend API
  - отображение результатов вычислений и ошибок
- **Взаимодействие**:
  - использует `fetch` для вызова backend-эндпоинтов
  - работает с JSON-ответами в едином формате

---

### Backend

- **Технология**: Python + Flask
- **Ответственность**:
  - предоставляет REST API эндпоинты по пути `/api/*`
  - выполняет валидацию входных параметров
  - выполняет вычисления
  - возвращает ответы в унифицированном JSON-формате
- **Контракт API**:
  - все ответы имеют единую структуру:
  
    ```json
    {
      "success": true,
      "data": {
        "result": 5
      },
      "error": null
    }
    ```

---

### CI/CD

- **Инструмент**: GitHub Actions
- **Пайплайны**:
  - запуск тестов backend
  - сборка Docker-образа
  - публикация Docker-образа
  - триггер деплоя в Render
  - сборка и деплой документации (MkDocs + GitHub Pages)

---

### Документация

- **Подход**: Documentation as Code
- **Инструменты**:
  - Markdown
  - MkDocs
  - markdownlint
- **Деплой**:
  - GitHub Pages
- **Расположение**:
  - директория `/docs` внутри репозитория

## Принципы проектирования

- простота важнее избыточной сложности
- явные и документированные API-контракты
- единый формат обработки ошибок
- документация является частью кодовой базы
- любое изменение API требует обновления документации
