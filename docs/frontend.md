# Frontend CI/CD Demo Project

## Назначение frontend-части

Frontend отвечает за пользовательский интерфейс приложения
и взаимодействие с backend API.

В рамках проекта frontend выполняет следующие задачи:

- отображение пользовательского интерфейса
- отправка HTTP-запросов к backend
- обработка успешных ответов и ошибок
- демонстрация работы API в реальном времени

Frontend является отдельным компонентом архитектуры
и взаимодействует с backend исключительно через HTTP API.

---

## Используемые технологии

Frontend реализован с использованием следующих технологий:

- **React**
- **Create React App**
- **JavaScript (ES6+)**
- **Fetch API**
- **HTML / CSS**

Выбор технологий обусловлен их распространённостью
и удобством для проектов.

---

## Архитектура frontend

Frontend построен по компонентному принципу.

Основные элементы:

- корневой компонент `App`
- функции для вызова backend API
- локальное состояние для результата и ошибок
- UI-элементы управления (кнопки)

Frontend не содержит бизнес-логики,
вся бизнес-логика находится на стороне backend.

---

## Взаимодействие с backend API

Frontend взаимодействует с backend через HTTP-запросы
к эндпоинтам вида:

/api/*

Пример запроса:

```bash
GET /api/add?a=2&b=3
```

Ответы backend обрабатываются в формате JSON
в соответствии с API-контрактом.

---

## Формат обработки ответов

Frontend ожидает следующие структуры:

### Успешный ответ

```json
{
  "success": true,
  "data": {
    "result": 5
  },
  "error": null
}
```

### Ответ с ошибкой

```json
{
  "success": false,
  "data": null,
  "error": {
    "message": "Описание ошибки"
  }
}
```

Frontend не делает предположений о структуре данных
и опирается только на задокументированный контракт.

## Конфигурация API URL

Адрес backend API настраивается через переменные окружения.

Используется переменная:

```!
REACT_APP_API_BASE
```

***Примеры значений**

Локальная разработка:

```!
REACT_APP_API_BASE=http://localhost:10000
```

Production:

```!
REACT_APP_API_BASE=https://myapp-latest-b5gs.onrender.com
```

Если переменная не задана,
frontend использует относительные пути /api/*.

## CORS

Frontend и backend могут находиться
на разных доменах и портах.

Для корректной работы frontend:

backend должен возвращать CORS-заголовки

CORS настраивается на стороне Flask через flask-cors

Без корректной настройки CORS
браузер будет блокировать запросы.

## Локальный запуск frontend

Для локального запуска frontend:

```bash
cd frontend
npm install
npm start
```

```http
http://localhost:3000
```

## Production-сборка frontend

Для production-сборки используется команда:

```bash
npm run build
```

В результате создаётся директория:

```bash
build/
```

Содержимое директории представляет собой
статические файлы,
готовые к деплою на Static Hosting.

## Деплой frontend (планируется)

Планируемая схема деплоя frontend:

сборка frontend через npm run build

деплой в Render как Static Site

отдельный URL для frontend

backend доступен через API URL
